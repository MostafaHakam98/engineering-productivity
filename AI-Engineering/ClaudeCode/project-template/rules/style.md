# Style Rules

## General

- Follow existing project style.
- Avoid drive-by formatting.
- Keep naming consistent with nearby code.
- Prefer explicit types where the project already uses them.
- Prefer small functions with clear boundaries.

## Reviews

For reviews, classify findings as:

- Blocker
- Important
- Suggestion
- Question

Each finding should include:

1. What is wrong.
2. Why it matters.
3. Suggested fix.
